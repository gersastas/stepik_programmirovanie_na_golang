### **Условная конструкция if**

Конструкция `if` проверяет условие — выражение, которое возвращает значение типа `bool`. Если условие истинно (равно `true`), выполняется блок инструкций, заключенный в фигурные скобки:

```go
package main

import "fmt"

func main() {
   a := 6    
   b := 7
   if a < b {     
      fmt.Println("a меньше, чем b")  
   }
}
                  
```

В этом примере условие `a < b` возвращает `true`, так как `a` действительно меньше `b`, и будет выполнена инструкция внутри блока `if`, выводящая на экран строку "a меньше, чем b".

### **If с краткой инструкцией**

Как и в цикле `for`, в конструкции `if` можно добавить инструкцию, которая будет выполнена до проверки самого условия. Эта инструкция выполняется один раз, перед тем как будет проверено условие, а переменные, объявленные в этой инструкции, доступны только внутри блока `if`.

Пример использования краткой инструкции в `if`:

```go
package main

import (
    "fmt"
    "math"
)

func main() {
    x := 2
    n := 3
    lim := 10

    // В инструкции перед условием вычисляется значение v
    if v := math.Pow(float64(x), float64(n)); v < float64(lim) {
        fmt.Printf("Результат: %.2f меньше лимита\n", v)
    }
}

                  
```

В данном примере переменная `v` объявляется и инициализируется перед проверкой условия `v < lim`. Значение `v` вычисляется как `x^n` (то есть 2 в степени 3, что дает 8), и так как `8 < 10`, условие истинно, и будет выведено сообщение "Результат: 8.00 меньше лимита".

Переменная `v` существует только в пределах блока `if`, и ее нельзя использовать за его пределами.

### **Условные конструкции else if и else**

Если нам нужно проверить несколько условий, мы можем использовать оператор `else if`:

```go
if a < b {
    fmt.Println("a меньше b")
} else if a > b {
    fmt.Println("a больше b")
}
                  
```

Если нам нужен вариант, когда ни одно из условий не выполняется, то мы используем оператор `else`:

```go
if a < b {
    fmt.Println("a меньше b")
} else if a > b {
    fmt.Println("a больше b")
} else {
    fmt.Println("a равно b")
}
```

### **Switch**

Предположим, мы хотим написать программу, которая выводит английские названия чисел. Без использования оператора `switch` программа могла бы выглядеть следующим образом:

```go
if i == 0 { 
    fmt.Println("Zero") 
} else if i == 1 {
    fmt.Println("One")
} else if i == 2 {
    fmt.Println("Two")
} else if i == 3 {
    fmt.Println("Three")
} else if i == 4 { 
    fmt.Println("Four")
} else if i == 5 {
    fmt.Println("Five")
}
                  
```

Этот код довольно громоздкий. Вместо такого подхода можно использовать оператор `switch`, который позволяет сделать программу более компактной:

```go
switch i {
case 0: fmt.Println("Zero")
case 1: fmt.Println("One")
case 2: fmt.Println("Two")
case 3: fmt.Println("Three")
case 4: fmt.Println("Four")
case 5: fmt.Println("Five")
default: fmt.Println("Unknown Number")
}
                  
```

Оператор `switch` начинается с ключевого слова `switch`, за которым идет выражение (в данном случае `i`). Далее идут блоки `case`, которые содержат возможные значения. Значение выражения сравнивается с каждым значением после `case`, и если совпадение найдено, выполняется соответствующий блок кода. Если ни одно из значений не совпало, выполняется блок `default`, аналогичный блоку `else` в условных операторах.

Обратите внимание, что в **switch** можно использовать любой тип данных.

### **Стоит дополнить, что:**

1. В Go код внутри блока `case` выполняется до следующего `case`, и нет необходимости заканчивать каждый блок `case` ключевым словом `break`, что помогает избежать ошибок. Однако если в блоке `case` используется ключевое слово `fallthrough`, то выполнение перейдет к следующему `case`, даже если его условие не истинно. Пример:

```go
v := 42
switch v {
case 100:
    fmt.Println(100)
    fallthrough
case 42:
    fmt.Println(42)
    fallthrough
case 1:
    fmt.Println(1)
    fallthrough
default:
    fmt.Println("default")
}
// Вывод:
// 42
// 1
// default
                  
```

В этом примере условие первого блока не выполняется, поэтому блок с `case 100` будет пропущен. Условие следующего блока будет равно `true`, поэтому выполнится содержащийся в нем код (`fmt.Println(42)`). И также сработает содержащийся в блоке `fallthrough` благодаря чему будет выполнен следующий блок (`case 1`) даже без проверки условия. Похожая ситуация происходит и с блоком `case 1`, где код продолжит выполнение далее, "проваливаясь" в блок `default` и выполняя код из него.

2. Также в Go существует специальная форма `switch`, где можно использовать произвольные условия в каждом блоке `case`. Это позволяет сделать код еще более гибким:

```go
var c uint32
fmt.Scan(&c)
switch {
case 1 <= c && c <= 9:
    fmt.Println("от 1 до 9")
case 100 <= c && c <= 250:
    fmt.Println("от 100 до 250")
case 1000 <= c && c <= 6000:
    fmt.Println("от 1000 до 6000")
}
                  
```

Здесь не требуется указание переменной для сравнения сразу после `switch`. После каждого `case` идет произвольное условие, которое будет проверяться. Этот подход делает `switch` удобным для использования в различных ситуациях.
