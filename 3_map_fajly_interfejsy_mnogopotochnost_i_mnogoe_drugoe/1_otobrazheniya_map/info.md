# Отображения (или карты) в Go

Отображения (или карты) — это структура данных, представляющая собой неупорядоченную коллекцию пар "ключ-значение". Важно, что каждый ключ в отображении уникален, а связанное с ним значение можно получить, обновить или удалить независимо от других элементов. Это делает отображения полезным инструментом для хранения и работы с данными, где важен быстрый доступ по ключу.

В языке Go отображение представляет собой ссылку на хеш-таблицу, и его тип записывается как `map[K]V`, где `K` — тип ключа, а `V` — тип значения. Все ключи в одном отображении должны иметь одинаковый тип, как и все значения. Однако тип ключа не обязан совпадать с типом значения. Главное, что ключи должны быть сравниваемыми с помощью оператора `==`, чтобы Go мог проверять их на равенство.

## Как создать отображение?

Для объявления отображения в Go используется следующий синтаксис:

```go
var users map[string]int
// где string - тип ключа, а int - тип значений
                  
```

**Важно! Этот способ опасен.** Если не инициализировать отображение, при попытке добавления значений программа вызовет ошибку. (см. пример ниже)

Для безопасного создания отображений существует два правильных способа:

```go
// С использованием встроенной функции make:
m1 := make(map[int]int)

// Или с использованием литерала отображения:
m2 := map[int]int{
    12: 2,   // ключ 12, значение 2
    1:  5,   // ключ 1, значение 5
}

fmt.Println(m1) // вывод: map[]
fmt.Println(m2) // вывод: map[1:5 12:2]
                  
```

Почему нельзя просто объявить отображение с помощью ключевого слова `var`? Если мы объявляем переменную с использованием `var` и не присваиваем ей начальное значение, Go присваивает переменной нулевое значение. Для отображения это будет значение `nil`, а попытка добавить элемент в `nil` отображение приведет к ошибке **assignment to entry in nil map** и панике:

```go
var m map[int]int
m[12] = 3
fmt.Println(m)

// Вывод (сработает паника):
// panic: assignment to entry in nil map
                  
```

Конечно, такой подход можно использовать, если результатом функции будет отображение, но нужно быть осторожным с такими случаями.

Вот два безопасных варианта:

```go
var m map[int]int
m = someFunc() // someFunc возвращает отображение
fmt.Println(m)
// Альтернативный вариант
var m = map[int]int{} // Инициализация пустого отображения
m = someFunc()        // someFunc возвращает отображение
fmt.Println(m)
                  
```

Как видите, второй вариант гораздо безопаснее, так как он гарантирует, что переменная `m` всегда будет корректно инициализирована.

## Работа с отображениями в Go

Для обращения к элементам отображения в Go используется обычная индексация. Например, чтобы получить значение по ключу, можно сделать так:

```go
m := map[int]int{
	12: 2,
	1:  5,
}

fmt.Println(m[12]) // 2
                  
```

Удаление элементов осуществляется с помощью встроенной функции `delete`. Она удаляет элемент по указанному ключу:

```go
m := map[int]int{
	12: 2,
	1:  5,
}

delete(m, 12) // Удаляем элемент с ключом 12
fmt.Println(m) // map[1:5]
                  
```

Важно, что операции обращения и удаления в отображении безопасны, даже если ключ не существует. Если мы попытаемся обратиться к ключу, которого нет в отображении, то Go вернет нулевое значение соответствующего типа:

```go
m := make(map[int]int)

fmt.Println(m[12]) // 0 (по умолчанию int = 0)

delete(m, 12)
fmt.Println(m) // map[]
                  
```

В примере выше, при отсутствии ключа, возвращается нулевое значение для типа, в данном случае `0` для `int`. Это поведение может стать причиной ошибок, если нулевое значение не обрабатывается должным образом. Чтобы избежать недоразумений, можно использовать проверку наличия ключа в отображении:

```go
m := map[int]int{
	1: 10,
}

if value, inMap := m[1]; inMap {
	fmt.Println(value) // 10 (ключ 1 присутствует)
}

if value, inMap := m[2]; inMap {
	fmt.Println(value) // Условие не выполнится
}
                  
```

Второе значение в данной проверке — это логическая переменная `inMap`, которая сообщает, существует ли такой ключ в отображении. Это значение часто называют `ok` для удобства.

Для перебора всех пар "ключ-значение" в отображении используется цикл с `range`. Цикл будет перебирать все элементы и на каждой итерации присваивать переменным `key` и `value` значения текущей пары:

```go
for key, value := range m {
    fmt.Println(key, value)
}
                  
```

Кроме того, Go позволяет использовать функцию `len` для определения количества пар "ключ-значение", хранящихся в отображении:

```go
m := map[int]int{
	1: 10,
	2: 20,
	3: 30,
}
fmt.Println(len(m)) // 3
```
